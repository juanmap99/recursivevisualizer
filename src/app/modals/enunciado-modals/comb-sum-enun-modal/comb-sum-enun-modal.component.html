Introducción problema

Se otorgaran dos parametros:

  • Array elementos: Array compuesto por valores numéricos enteros
  • Target sum: Valor numérico que representa la target

Se pide el diseño de un algoritmo que retorne la cantidad de combinaciones a través de las cuales obtenemos como resultado Target sum, en donde no es requisito utilizar todos los elementos del array, cada elemento puede usarse todas las veces que se lo considere necesario y la única operación válida es la suma.

Asumir que todos los números del array son positivos.

Ejemplos

Ejemplo A

  • Parametros
    • Array elementos: [1,2,3]
    • Target sum: 4
  • Respuesta: 7
  • Combinaciones posibles
    • Comb.1 = 1 + 1 + 1 + 1 = 4
    • Comb.2 = 1 + 1 + 2 = 4
    • Comb.3 = 1 + 2 + 1 = 4
    • Comb.4 = 2 + 1 + 1 = 4
    • Comb.5 = 3 + 1 = 4
    • Comb.6 = 1 + 3 = 4
    • Comb.7 = 2 + 2 = 4

Ejemplo B

  • Parámetros
    • Array elementos: [9]
    • Target sum: 3
  • Respuesta: 0

Restricciones

Las restricciones que se le impondrán a la solución son las siguientes:

  • Resolver el problema utilizando recursividad.
  • Realizar dos soluciones, una que no utilice programación dinámica, y otra que si.

Follow-up

Se presentan los siguientes desafios como follow-up al problema:

  • Analice como se ve afectada la complejidad temporal del algoritmo al pasar de una solución que no utiliza programación dinámica hacia una que si.
  • Dibuje un árbol que represente las llamadas recursivas que se realizarán en el cálculo cuando el array es [1,2,3] y el target es 4 tanto para el caso en donde no se utiliza programación dinámica como para el caso en el que si.
  • Corra el código a través de la herramienta bajo el modo de funcionamiento que considere conveniente y analice y compare el output de respuesta con sus predicciones.
