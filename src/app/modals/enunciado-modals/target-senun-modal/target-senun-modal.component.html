Introducción problema

Se otorgaran dos parametros:

  • Array elementos: Array compuesto por valores numéricos enteros
  • Target sum: Valor numérico que representa la target

Se pide el diseño de un algoritmo que retorne la cantidad de ecuaciones que se pueden construir utilizando todos los elementos del array de elementos que den como resultado Target sum, en donde cada elemento debe ser utilizado solo una sola vez y las únicas operaciones válidas son suma y resta.

Asumir que todos los números del array son positivos.

Ejemplos

Ejemplo A

  • Parámetros
    • Array elementos: [1,2,4,2]
    • Target sum: 5
  • Respuesta: 2
  • Ecuaciones posibles
    • Ec.1 = 1 + 2 + 4 - 2 = 5
    • Ec.2 = 1 - 2 + 4 + 2 = 5

Ejemplo B

  • Parámetros
    • Array elementos: [1,1,1,1,1]
    • Target sum: 3
  • Respuesta: 5
  • Ecuaciones posibles
    • Ec.1 = -1 + 1 + 1 + 1 + 1 = 3
    • Ec.2 = +1 - 1 + 1 + 1 + 1 = 3
    • Ec.3 = +1 + 1 - 1 + 1 + 1 = 3
    • Ec.4 = +1 + 1 + 1 - 1 + 1 = 3
    • Ec.5 = +1 + 1 + 1 + 1 - 1 = 3

Restricciones

Las restricciones que se le impondrán a la solución son las siguientes:

  • Resolver el problema utilizando recursividad.
  • Realizar dos soluciones, una que no utilice programación dinámica, y otra que si.

Follow-up

Se presentan los siguientes desafios como follow-up al problema:

  • Analice como se ve afectada la complejidad temporal del algoritmo al pasar de una solución que no utiliza programación dinámica hacia una que si.
  • Dibuje un árbol que represente las llamadas recursivas que se realizarán en el cálculo en donde el array es [1,1,1,1] y el target es 2 tanto para el caso en donde no se utiliza programación dinámica como para el caso en el que si.
  • Corra el código a través de la herramienta bajo el modo de funcionamiento que considere conveniente y analice y compare el output de respuesta con sus predicciones.
